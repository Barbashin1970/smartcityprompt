# Промпт-конструктор для аналитиков умного города

Легковесное браузерное приложение (SPA) для быстрой генерации структурированных промптов к большим языковым моделям (LLM). Инструмент создан специально для аналитиков, работающих с городскими данными: транспортом, экологией (PM2.5, AQI), отключениями ЖКХ и муниципальными реестрами. Он основан на подходе с полностью статическим приложением: один HTML-файл, который можно открыть локально или задеплоить как обычный статический сайт.

## 🚀 Особенности

- **Zero-Backend (BYOM)**: Приложение полностью работает в браузере (HTML/JS/CSS). Не требует API-ключей, бэкенда и не отправляет данные на сторонние серверы. Вы копируете готовый промпт и вставляете в любую удобную нейросеть (ChatGPT, GigaChat, YandexGPT) или локальную модель в LM-Studio.
- **Специализация под умный город**: Преднастроенные роли (эколог, транспортник, аналитик ЖКХ) и источники данных (opendata, 051.novo-sibirsk.ru, Open-Meteo, DuckDB).
- **Text-to-SQL поддержка**: Специальный блок для инжекта схем баз данных (DDL/YAML) для точной генерации SQL-запросов (например, DuckDB-совместимых).
- **Мультиформатность**: Экспорт промптов в классическом виде `:::`, в `XML-тегах` (идеально для локальных LLM) и структурированном `Markdown`.
- **Локальные пресеты**: Возможность сохранять любимые шаблоны промптов в браузере (`localStorage`).

## 🛠 Установка и запуск

Поскольку проект состоит из статических файлов, запустить его максимально просто:

### Способ 1: Локальный запуск
Просто скачайте репозиторий и откройте файл `index.html` в любом современном браузере.

### Способ 2: Бесплатный хостинг (Vercel / Railway / GitHub Pages)
В корне проекта лежит конфигурация `vercel.json` для мгновенного развертывания статики:
1. Привяжите репозиторий к Vercel или Railway.
2. Проект автоматически развернется и будет доступен по публичной ссылке.

## 💡 Как пользоваться

1. **Выберите роль и задачу**: Например, «Муниципальный аналитик» и «Найди закономерности и тренды».
2. **Укажите источники**: Опишите, откуда взяты данные и в каком формате вы хотите получить ответ (например, JSON или таблица).
3. **Опишите структуру (опционально)**: Если нужен SQL-запрос, вставьте в поле схемы описание вашей таблицы из `canonicalschemas.yaml`.
4. **Настройте качество**: Укажите уровень детализации, формальность и объем с помощью слайдеров.
5. **Скопируйте результат**: Нажмите «Копировать весь промпт» или скопируйте SYSTEM и USER части раздельно для LM-Studio.

## ⚙️ Технологический стек

- **HTML5 / CSS3 / Vanilla JS** — никаких тяжелых фреймворков и сборщиков.
- **Clipboard API** — для надежного копирования в буфер обмена.
- **localStorage** — для безопасного хранения пользовательских пресетов без базы данных.
